# How to Request Changes - Teacher Guide

## Overview

As teachers at Mergington High School, you don't need to modify the activities system code directly. Instead, you can use our structured issue templates to request changes. These templates help ensure that our GitHub Copilot coding agent has all the information needed to implement your requests quickly and accurately.

## Available Issue Templates

### 📚 Activity Management
**Use this for:** Adding new activities, modifying existing activities, changing schedules, updating capacity limits, or removing activities.

**Examples:**
- "Add a new Robotics Club that meets on Wednesdays"
- "Change Chess Club from Mondays to Tuesdays"
- "Increase Art Club capacity from 15 to 20 students"

### 🐛 Bug Report
**Use this for:** Reporting problems or things that aren't working correctly.

**Examples:**
- "Students can't register for Drama Club"
- "Activity schedule shows wrong times"
- "Website doesn't work on mobile phones"

### ✨ Feature Request
**Use this for:** Requesting completely new functionality that doesn't exist yet.

**Examples:**
- "Add ability to export student contact lists"
- "Send email notifications to parents"
- "Create attendance tracking"

### 🎨 UI/UX Improvement
**Use this for:** Making the website easier to use or better looking.

**Examples:**
- "Make activity cards larger on mobile"
- "Change button colors to school colors"
- "Reorganize the page layout"

### 📊 Data Management
**Use this for:** Changing what information is collected, how it's displayed, or how it can be exported.

**Examples:**
- "Add phone number fields for students"
- "Export student lists to Excel"
- "Add grade level information"

## How to Create an Issue

1. **Go to the Issues tab** in this GitHub repository
2. **Click "New Issue"**
3. **Choose the appropriate template** from the list
4. **Fill out all required fields** - be as detailed as possible
5. **Click "Submit new issue"**

## Tips for Better Results

### Be Specific
- Instead of: "Make the website better"
- Try: "Make activity descriptions easier to read on mobile phones by increasing font size"

### Provide Context
- Explain why you need the change
- Describe how it will help your work
- Mention any school policies or requirements

### Include Examples
- If requesting new features, describe similar functionality you've seen elsewhere
- If reporting bugs, provide exact steps to reproduce the problem
- If requesting design changes, reference examples of what you like

### Use Clear Language
- Write as if explaining to a colleague who isn't familiar with your daily work
- Avoid technical jargon unless necessary
- Be precise about names, dates, and requirements

## What Happens Next

1. **Your issue is reviewed** by our development team
2. **GitHub Copilot coding agent** analyzes your request
3. **Code changes are implemented** based on your specifications
4. **You'll be notified** when the changes are ready for testing
5. **You can verify** that the changes meet your needs

## Getting Help

If you're unsure which template to use or need help filling out a request:

- **Development Guide**: Check our [development documentation](../docs/how-to-develop.md)
- **Ask Questions**: Use GitHub Discussions for general questions
- **Contact IT**: Reach out to the school's IT support team

## Template Quality Standards

Our templates are designed to ensure that every request includes:

- ✅ **Clear problem description** - What needs to change and why
- ✅ **Detailed requirements** - Exactly what you want the result to look like
- ✅ **Acceptance criteria** - How we'll know the request is complete
- ✅ **Context and constraints** - Important limitations or requirements
- ✅ **Priority information** - How urgent the request is

This structured approach helps our Copilot coding agent understand and implement your requests without needing to ask follow-up questions, making the process faster and more efficient for everyone.
