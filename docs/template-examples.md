# Issue Template Examples

This document shows examples of how teachers would use the issue templates to request changes.

## Example 1: Activity Management Request

**Template Used:** Activity Management  
**Scenario:** Ms. Rodriguez wants to add a new Art Therapy club

```
Type of Request: Add a new activity
Activity Name: Art Therapy Club
Desired Changes: Create a new activity for students interested in therapeutic art practices
Schedule Details: Thursdays, 3:30 PM - 5:00 PM
Maximum Participants: 10
Reason: Several students have requested art-based stress relief activities
Priority: Medium - Needed within a few weeks
```

## Example 2: Bug Report

**Template Used:** Bug Report  
**Scenario:** Mr. Chen found that students can't register for Chess Club

```
Problem Description: When students try to register for Chess Club, they get an error message and aren't added to the participant list
Affected Area: Student registration
Steps to Reproduce:
1. Go to activities page
2. Click on "Chess Club"  
3. Click "Register Student"
4. Enter email: student@mergington.edu
5. Click submit
6. See error message
Frequency: Always
Impact Level: High - Major disruption to daily work
```

## Example 3: Feature Request

**Template Used:** Feature Request  
**Scenario:** Principal Martinez wants to export contact lists

```
Feature Description: Export student contact information for each activity to Excel files
Current Problem: Need to manually copy student emails for parent communications
Feature Category: Data export/reporting
Detailed Requirements: Button on each activity page to download CSV with student names, emails, phone numbers
User Story: As an administrator, I want to export contact lists so I can quickly send announcements to parents
Priority Level: High - Would significantly improve efficiency
```

## Example 4: UI/UX Improvement

**Template Used:** UI/UX Improvement  
**Scenario:** Students complain that activity cards are hard to read on phones

```
Improvement Type: Mobile device display
Current Situation: Activity cards are too small and cramped on mobile phones
Desired Improvement: Make cards larger with bigger fonts and more spacing
Affected Users: Students using mobile phones
Device Focus: Mobile phones primarily
Priority Level: Medium - Would improve user experience noticeably
```

## Example 5: Data Management

**Template Used:** Data Management  
**Scenario:** Need to add emergency contact information

```
Data Request Type: Add new information fields
Data Type: Student information
Current Situation: Only have student emails, need emergency contacts
Desired Changes: Add phone number and emergency contact fields to student registration
Data Privacy Level: Limited - Only teachers should see this information
Business Justification: School policy requires emergency contacts for all extracurricular participants
Urgency Level: High - Needed within 2 weeks
```

These examples show how the structured templates guide teachers to provide all necessary information for GitHub Copilot to implement their requests accurately.
